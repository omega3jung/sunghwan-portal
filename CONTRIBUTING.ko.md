# 브랜치 및 PR 접두사 규칙

이 프로젝트는 다음과 같은 간소화된 접두사 규칙을 따릅니다.

- feat: 기능 개발
- fix: 버그 수정 (핫픽스 포함)
- refactor: 내부 리팩토링
- docs: 문서 변경
- chore: 툴 및 구성 변경
- test: 테스트 관련 변경 (단위, 통합 또는 UI 테스트)

규칙을 예측 가능하고 유지 관리하기 쉽게 유지하기 위해 의도적으로 최소한의 접두사만 사용합니다.

PR 제목은 다음 형식을 따라야 합니다.

```txt
<유형>(<범위>): <설명>
```

---

## 유형

- feat: 새로운 기능
- fix: 버그 수정
- refactor: 코드 리팩토링
- docs: 문서 변경
- chore: 툴 또는 구성 변경
- test: 테스트 추가 또는 업데이트

---

### 예시

```md
feat(auth, ui): 가장 인증 구현 및 홈 UI 완성
```

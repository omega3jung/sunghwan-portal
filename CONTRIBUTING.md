This project uses a simplified conventional prefix strategy:

- feat: feature development
- fix: bug fixes (including hotfixes)
- refactor: internal refactoring
- docs: documentation changes
- chore: tooling and configuration
- test: test-related changes (unit, integration, or UI tests)

The set is intentionally minimal to keep conventions predictable and easy to maintain.

PR titles must follow this format:

```txt
<type>(<scope>): <description>
```

---

## Types

- feat: New feature
- fix: Bug fix
- refactor: Code refactoring
- docs: Documentation changes
- chore: Tooling or configuration changes
- test: Adding or updating tests

---

### Example

```md
feat(auth, ui): implement impersonation authentication and complete home UI
```

# 📁 src Folder Structure

This document describes the folder structure and purpose of each directory under the `src` directory.

---

<details>
  <summary>English</summary>

# app/ - Next.js App Router root.

# app/home/ - Demo home pages.

# app/home/demo/ - Demo pages for custom components.

# app/home/service-desk/ - Main demo: list view, charts, and visualization.

# app/login/ - Demo login pages using the test DB.

# components/ - Components root.

# components/ui/ - Shadcn UI components.

# components/custom/ - Custom UI components.

# components/layout/ - Page and layout-level components.

# hooks/ - Global reusable hooks.

# lib/ - API client, server utilities, external library wrappers, data parsers, etc.

# lib/i18n/ - Localization resources for pages.

# - Custom component-specific locales are placed in each component folder.

# lib/api/ - Fetch calls.

# lib/auth/ - Next-auth config.

# lib/db/ - Prisma + repository layer, PlanetScale.

# lib/schemas/ - Zod validations.

# lib/utils/ - Utility library.

# public/ - Static files: fonts, icons, images, metadata, static JSON, PWA assets, etc.

# stories/ - Stories for common components.

# - Stories for custom components are stored inside each custom component folder.

# styles/ - Global and shared CSS files.

# test/ - Jest test cases.

# - Tests for custom components are stored inside each custom component folder.

# types/ - TypeScript definitions for this project.

# types/api/ - API-related types (request, response, payload).

# types/common/ - Common and shared custom types.

# utils/ - Utility functions used across this project.

</details>

## Repository Structure Strategy

- 현재 포트폴리오 데모는 Next.js(App Router) 기반 단일 Repository로 구축함.
- 인증(next-auth), ORM(Prisma), DB(MySQL/PlanetScale)까지 모든 계층을 한 Repo에서 관리.
- 프론트엔드 중심 포지션이므로 단일 Repo 구성은 단순하고 평가자에게도 명확함.
- 다만 향후 실무형 구조(Front-end / Back-end 분리)로 확장 가능한 형태로 디렉토리를 설계함.

### lib/ 디렉토리 구조 원칙

- `lib/api`: 프론트엔드에서 REST API 호출을 모아두는 fetch wrapper 계층.
- `lib/db`: Prisma와 Repository Layer. 서버 전용.
- `lib/auth`: next-auth 설정. 서버 전용.
- `lib/i18n`: 다국어 리소스.
- `lib/schemas`: Zod 스키마 (Form + API 공용).
- `lib/utils`: 재사용 가능한 유틸리티 모음.

### Future Scalability

- 프론트/백엔드 분리 시 `lib/db`와 `app/api`를 그대로 백엔드 Repo로 이동 가능.
- 프론트는 `lib/api`의 endpoint URL만 변경하면 그대로 재사용 가능.
- 인증(next-auth)도 독립 가능하도록 config 파일 기반으로 구성함.

현재는 단일 Repo 유지 → 추후 확장성 확보하는 방향으로 개발을 진행함.
